# AGENTS.md

This is a **LaTeX resume project** (fork of Deedy-Resume-For-Chinese), not a software app. There is no test/lint/typecheck pipeline — verification means recompiling the PDF.

**After every content edit to a `resume.tex`, recompile the affected PDF before considering the task done.** A successful edit is one that still compiles to a single page.

## Build

- `make` (documented) compiles both variants via `scripts/build.sh`. **On this Windows machine `make` is not installed** and WSL `bash` cannot see the Windows TeX Live PATH, so `build.sh` fails. Use the direct command below instead.
- Compile the Chinese resume (the primary file users edit):
  ```
  & "D:\texlive\2024\bin\windows\xelatex.exe" -interaction=nonstopmode -halt-on-error resume.tex
  # then move OpenFonts.Chinese\resume.pdf -> docs\resume-cn.pdf
  ```
  Run from the variant dir (e.g. `OpenFonts.Chinese\`) so `fonts/` is on the lookup path.
- **Must use XeLaTeX**, not pdflatex.
- BibTeX is only needed if the commented-out `\bibliography{publications}` block in `resume.tex` is re-enabled.
- Expected font-shape warnings (small-caps substitution for 思源宋体) are normal and harmless; treat only `Error`/`! ` lines as failures.
- Success criterion: `Output written on resume.pdf (1 page)`. If it reports 2+ pages, the edit caused overflow — trim it.
- Build outputs: `docs/resume-cn.pdf` (Chinese), `docs/resume.pdf` (English).

## Directory layout — what to edit

- `OpenFonts.Chinese/` — **the user's actual resume** (Chinese, primary). `resume.tex` is the entrypoint; `deedy-resume-openfont.cls` holds section/link macros; `fonts/` bundles 思源黑体 + 思源宋体. Edit content here.
- `OpenFonts/` — English variant. `resume.tex` still contains **upstream sample data** (Gao Ce), not the user's. Do not treat its content as ground truth; only edit if asked to produce an English version.
- `MacFonts/` — **unmaintained** per `README.md`. Do not modify.
- `docs/` — build output only (committed PDFs). regenerated by `make`.
- `scripts/build.sh` — single build script; invoked by the Makefile.
- `.workbuddy/memory/` — long-term user context (name, education, work history, skill levels). Written in Chinese. Useful for filling placeholders consistently; not part of the build.

## Compile quirks

- Both `resume.tex` files end with `\end{document}  \documentclass[]{article}` on the same line. This is an upstream template quirk; harmless because TeX stops at `\end{document}`. Do not "fix" it.
- The `.cls` redefines `\refname` and provides `\lastupdated`, `\namesection`, `\runsubsection`, `\descript`, `\location`, `\sectionsep`, `\tightemize`. Use these macros when adding sections — do not introduce raw `\section*` styling.
- Resume is designed to fit **one page**. Adding content to either column can overflow; keep entries tight.
- Font lookups depend on the bundled `fonts/` subdir being present next to the `.tex` file. Missing-font errors usually mean the build was run from the wrong working directory (`build.sh` `cd`s into each variant dir for this reason).

## Editing conventions

- Content language for `OpenFonts.Chinese/resume.tex` is Chinese; section headers and skill-level labels (熟练/熟悉/了解) are established — match them.
- Personal data lives inline in `resume.tex` (`\namesection{张靖昊}{}{...}`); cross-check against `.workbuddy/memory/MEMORY.md` before editing identity/contact fields.
- Placeholder strings like `待补充` mark fields the user has not yet filled (e.g. 项目经历, 所获荣誉). Preserve them as placeholders rather than inventing content.
